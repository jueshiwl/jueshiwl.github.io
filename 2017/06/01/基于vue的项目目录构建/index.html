基于vue的项目目录构建

发表于 2017-06-01

其实只要是项目都会涉及到项目目录构建的问题，因为工作都是在别人构建好的项目目录上开发，没有考虑过这个问题。写的demo也基本就是vue-cli里面的目录稍微调整下，但是当我看了一些别人的源码后，我才发先项目目录的构建也是一个重要的东西。

  • src
    • assets 存放需要webpack打包的静态文件
      • fonts 字体文件
      • images 图片
      • js 一些外部的js库
      • scss 一些通用的css
    • components 存放项目中一些通用的组件
    • router 存放页面路由
      • index.js 路由总入口
      • xxx.js 相关模块的路由配置
      • …
    • mock 存放mock数据(前后端分离mock数据使用，按需求增删)
      • xxx
    • vuex vuex(需要使用vuex的时候加入)
      • action.js 处理methods的dispatch过来的名称与之对应，并提交到mutations
      • getter.js 存放共用的 Computed Getters，实现在多个组件之间的共享
      • store.js 存放把modules文件下的多个模块集合在一起来创建 Vuex 实例,并导出 store
      • mutations-type.js 定义总的不同事件的类型，方便modules文件中的模块去用
      • modules 存放不同的模块功能区，里面可以有自己的actions、mutations，store等
    • services 存放所有请求
      • xhr.js 封装好的http方法
      • xxx.js 相关模块的数据发送/请求
      • …
    • views 页面层
      • index 每一个页面作为单独的组件
        • xxx
    • App.vue 项目的入口组件
    • main.js 项目的入口文件(引入一些通用的库，全局配置)

一直都是我原来一直没有想过项目目录应该怎么构建，哪些东西需要抽离出来，哪些是可以按层级目录
